AUSTRALIAN CURRICULUM v9 — CONTENT DESCRIPTORS

Year 2

CodeDescriptor
**AC9M2N01**Quantify sets of objects to at least 1000; count collections to 1000 by partitioning numbers using place value
**AC9M2N02**Add and subtract numbers to 1000 using efficient mental and written strategies and choose appropriate computational choices
**AC9M2N03**Solve problems involving addition and subtraction using efficient mental and written strategies and non-digital tools, interpreting results in context

General Capabilities (embedded throughout)

  • Numeracy — core
  • Literacy — problem-solving language, explanation writing
  • Critical & Creative Thinking — strategy selection, justification

    • BIG UNDERSTANDINGS

    Students will understand that:

  • Adding and subtracting with larger numbers relies on place value — tens and hundreds
  • Efficient strategies (jump, split, compensation) work for big numbers too
  • Estimation before calculating helps catch errors
  • Addition and subtraction are inverse operations — always check with the opposite
  • Real-world problems can be solved more efficiently with written strategies
  • Mental and written strategies are both valid tools

    • ESSENTIAL QUESTIONS

  • How do we add and subtract efficiently with numbers to 1000?
  • What strategy works best for different types of problems?
  • How do we use place value to make computation easier?
  • Why should we estimate before we calculate?
  • How do we know our answer is right?

    • VOCABULARY — TIERED

    Tier 1 (everyday)

    add, subtract, plus, minus, total, difference, estimate, roughly, close to, check, correct, wrong

    Tier 2 (academic)

    place value, tens, hundreds, ones, regroup, jump strategy, split strategy, compensation, efficient, strategy

    Tier 3 (mathematical)

    mental computation, written strategy, column addition, column subtraction, trading, compatible numbers, inverse operation, reasonableness

    • DIFFERENTIATION FRAMEWORK

    EAL/D Students

  • Pre-teach mathematical vocabulary with visual dictionary cards
  • Sentence frames for strategy explanation: "I used the ___ strategy because ___."
  • Bilingual anchor charts for place value (hundreds, tens, ones) in home language
  • Accept oral explanation in place of written where appropriate
  • Allow additional processing time
  • Reduce written output — focus on demonstration with MAB blocks

    • Students Identified as Gifted/Advanced

  • Explore addition and subtraction with 4-digit numbers
  • Investigate: can you add two numbers without writing anything down? How?
  • Challenge: create a word problem with a two-step solution
  • Open investigation: what happens when you add 999 + 1? Can you explain using place value?
  • Introduce: addition of decimals (money: $4.50 + $3.25)

    • Students Not Yet at Expected Level (NEP)

  • Consolidate place value to 100 before introducing 3-digit numbers
  • Use MAB blocks for all computation — do not remove manipulatives too early
  • Focus on 2-digit + 2-digit without regrouping; extend to regrouping only when solid
  • Pre-lesson micro-session: number talk on bonds to 100
  • Peer support pairing
  • Modified success criteria: focus on 4-5 criteria

    LESSON 1.2 — Place Value to 100

    Duration: 50 minutes

    Type: Concept Introduction (I Do → We Do)

    Resources: MAB blocks, place value charts, number cards

    Learning Intentions

  • I am learning to read, write, and understand numbers to 100
  • I am learning that tens and ones are the building blocks of 2-digit numbers

    • Success Criteria

  • I can explain what the tens digit and ones digit mean in any 2-digit number
  • I can build the same number in different ways

    • I DO (15 min)

    Write 73 on the board.

    "7 tens and 3 ones. 70 + 3 = 73."
    "The 7 is not worth 7 — it's worth 70. Because it's in the TENS place."

    Build with MAB blocks: 7 rods (tens) + 3 cubes (ones).

    "Different ways to make 73:"
    - 7 tens + 3 ones
    - 6 tens + 13 ones (swap one rod for 10 cubes)
    - 5 tens + 23 ones
    - 4 tens + 33 ones
    - 3 tens + 43 ones
    "The number stays the same — 73 — but the PARTITION changes. This is flexible place value."

    WE DO (20 min)

    Build and partition:

    Use MAB blocks. Show multiple partitions for:

  • 64 (6 tens + 4 ones, 5 tens + 14 ones, 4 tens + 24 ones, etc.)
  • 85
  • 47

    • Place value challenge:

    Given MAB blocks, write the number. Given a number, build it in two different ways.

    YOU DO (15 min)

  • Write the value of each digit in 58: ___ tens = ___ ___ ones = ___
  • Partition 72 in two different ways.
  • "True or false: 4 tens + 18 ones = 58. Explain."
  • EXTENSION: "How many tens are in 100? How many ones?"

    • Differentiation

  • EAL/D: Place value chart on desk; digit name vs. value confusion pre-taught
  • NEP: Partition within tens and ones only (no swapping); focus on 7 tens + 3 ones = 73
  • Gifted: Extension + "What is the largest number you can make with 9 tens and 9 ones? What about 10 tens + 0 ones?"

    • LESSON 1.3 — Adding and Subtracting 10s and 100s Mentally

    Duration: 50 minutes

    Type: Guided Practice

    Resources: 100 grid, MAB blocks, number lines

    Learning Intentions

  • I am learning to add and subtract 10s mentally
  • I am learning that the ones digit stays the same when you add/subtract 10

    • Success Criteria

  • I can add and subtract 10 from any 2-digit number mentally
  • I can explain why the ones digit doesn't change

    • I DO (15 min)

    "47 + 10. What's the pattern?"

    Build 47 with MAB. Add one rod (10). Count: 57.

    "The ones digit STAYS the same. 7 ones stays 7 ones. Only the tens changed: 4 tens became 5 tens."
    "47 + 10 = 57. 47 - 10 = 37."
    "This works for any 2-digit number: add 10, the tens go up by 1; subtract 10, the tens go down by 1."
    "And for 100: add 100, the hundreds go up by 1, tens and ones stay the same."

    WE DO (20 min)

    Mental 10s and 100s:

  • 36 + 10 = ___ 36 - 10 = ___
  • 58 + 10 = ___ 58 - 10 = ___
  • 73 + 20 = ___ (add 2 tens)
  • 91 - 30 = ___ (subtract 3 tens)
  • 250 + 100 = ___ 250 - 100 = ___

    • Partner activity: one partner gives +10/-10 problem, the other solves mentally. Swap. Then progress to +20, +30, -20, -30.

    YOU DO (15 min)

  • Solve: a) 45 + 10 = ___ b) 67 - 10 = ___ c) 89 + 20 = ___ d) 54 - 30 = ___
  • "When you add 10 to a number, which digit changes? Which stays the same?"
  • "What is 100 more than 347? What is 100 less?"
  • EXTENSION: "What is 10 more than 99? What about 10 less? Can you explain?"

    • Differentiation

  • EAL/D: 100 grid and number line available; "when adding 10, the tens digit goes up" written as reminder
  • NEP: +10 and -10 only until solid; use 100 grid as support
  • Gifted: Extension + "What is 1000 - 100? What is the pattern?"

    LESSON 2.1 — Jump Strategy for 2-Digit Addition

    Duration: 50 minutes

    Type: Explore

    Resources: Number lines to 100, MAB blocks, individual whiteboards

    Learning Intentions

  • I am learning to add 2-digit numbers using the jump strategy on a number line
  • I am learning to jump by tens, then by ones

    • Success Criteria

  • I can show 34 + 25 on a number line using jumps
  • I can solve 2-digit addition using jump strategy

    • I DO (15 min)

    "34 + 25. How can we solve this using the jump strategy?"

    Draw number line.

    "Option 1: Jump the ones first. 34 + 20 = 54, then +5 = 59."
    "Option 2: Jump the tens first. 34 + 20 = 54, then +5 = 59. SAME ANSWERS!"
    "The order of jumps doesn't matter — we get the same result!"
    "Jump tens first is USUALLY easier because we land on a friendly number (a multiple of 10) first."

    Write:

    ```

    34 + 25

    = 34 + 20 + 5

    = 54 + 5

    = 59

    ```

    WE DO (20 min)

    Jump strategy practice:

  • 27 + 14: 27 + 10 = 37, 37 + 4 = 41
  • 43 + 38: 43 + 30 = 73, 73 + 8 = 81
  • 56 + 27: 56 + 20 = 76, 76 + 7 = 83
  • 38 + 45: 38 + 40 = 78, 78 + 5 = 83

    • Partner activity: one partner draws the jumps, the other writes the number sentence. Swap.

    YOU DO (15 min)

  • Show 38 + 23 on the number line. Write the steps.
  • Solve using jump strategy: a) 47 + 16 = ___ b) 53 + 28 = ___ c) 29 + 34 = ___
  • "Which do you prefer — jumping ones first or tens first? Why?"
  • EXTENSION: "Show 38 + 45 using BOTH orders. Do you get the same answer?"

    • Differentiation

  • EAL/D: Number line 0-100 on desk; step-by-step template for recording jumps
  • NEP: Addends with minimal renaming (e.g. 34+22, 47+31); tens jump first only
  • Gifted: Extension + "Can you use jump strategy for 156 + 37?"

    • LESSON 2.2 — Split Strategy: Partition by Place Value

    Duration: 50 minutes

    Type: Explore

    Resources: MAB blocks, place value charts

    Learning Intentions

  • I am learning to add using the split strategy — add tens, then add ones
  • I am learning that 34 + 25 = (30+4) + (20+5) = (30+20) + (4+5) = 50 + 9 = 59

    • Success Criteria

  • I can split 2-digit numbers into tens and ones and add separately
  • I can explain why the split strategy works

    • I DO (15 min)

    "Here's another powerful strategy: SPLIT by place value."
    "34 + 25. Split both numbers: 34 = 30 + 4. 25 = 20 + 5."

    Write:

    ```

    34 + 25

    = (30 + 4) + (20 + 5)

    = (30 + 20) + (4 + 5)

    = 50 + 9

    = 59

    ```

    "We add the tens together: 30 + 20 = 50. Then add the ones: 4 + 5 = 9. Then combine: 50 + 9 = 59."
    "Same answer as the jump strategy. Different LOOK. Same MATHS."
    "The split strategy shows WHY addition works — because of place value!"

    WE DO (20 min)

    Split strategy practice:

  • 47 + 32: (40+7) + (30+2) = (40+30) + (7+2) = 70 + 9 = 79
  • 56 + 28: (50+6) + (20+8) = (50+20) + (6+8) = 70 + 14 = 84 ← tricky! 6+8=14
  • 38 + 45: (30+8) + (40+5) = (30+40) + (8+5) = 70 + 13 = 83
    • "Wait — 56 + 28 gave us 70 + 14. 14 is more than a ten! What do we do?"
    "70 + 14 = 84. We can leave it as 84, or regroup: 14 = 10 + 4. 70 + 10 = 80, 80 + 4 = 84."
    "This is called RENAMING or REGROUPING. It happens when the ones add to 10 or more."

    YOU DO (15 min)

  • Use split strategy to solve: a) 43 + 35 = ___ b) 27 + 46 = ___ c) 58 + 24 = ___
  • "For 27 + 46, what do the tens add to? What do the ones add to?"
  • "When does the split strategy need regrouping? Give an example."
  • EXTENSION: "Use split strategy for 156 + 237. What is different?"

    • Differentiation

  • EAL/D: Split strategy template with labelled boxes (tens, ones); MAB blocks available
  • NEP: Addends where ones sum ≤ 9 (no regrouping needed); 47 + 31, 56 + 23
  • Gifted: Extension + "Can you split a 3-digit number? 234 + 145 = ?"

    • LESSON 2.3 — Choosing Between Jump and Split

    Duration: 50 minutes

    Type: Guided Practice

    Resources: Strategy choice cards, problem sets, whiteboards

    Learning Intentions

  • I am learning to choose the best strategy for each addition problem
  • I am learning that both jump and split strategies are valid

    • Success Criteria

  • I can explain my strategy choice for different problems
  • I can solve 2-digit addition problems efficiently

    • I DO (12 min)

    "Now you know TWO strategies. When do you use which?"

    Decision chart:

    ProblemBest StrategyWhy
    47 + 25SplitEasy tens: 40+20=60, 7+5=12, 60+12=72
    38 + 19Jump38+20=58, 58-1=57 (compensation)
    56 + 27SplitRenaming: 50+20=70, 6+7=13, 70+13=83
    "Sometimes split is faster. Sometimes jump is faster. Look at the numbers FIRST. Then decide."

    WE DO (20 min)

    Strategy choice: Problem cards. Students choose strategy, solve, explain why.

  • 43 + 28 — jump or split?
  • 37 + 15 — jump or split?
  • 54 + 39 — jump or split?
  • 68 + 25 — jump or split?

    • YOU DO (15 min)

  • Choose a strategy and solve: a) 47 + 26 b) 58 + 34 c) 39 + 43
  • Write your strategy and reason for each.
  • "Create a problem where jump strategy is clearly better than split. Create one where split is better."
  • EXTENSION: "What about 68 + 47? Which strategy would YOU choose?"

    LESSON 3.1 — Compensation Strategy

    Duration: 50 minutes

    Type: Explore

    Resources: Number lines, MAB blocks

    Learning Intentions

  • I am learning to use the compensation strategy — round one number, then adjust
  • I am learning that 38 + 19 = 38 + 20 - 1

    • Success Criteria

  • I can use compensation to add efficiently (e.g. add extra, then subtract the extra)
  • I can explain why compensation works

    • I DO (15 min)

    "Here's a sneaky fast strategy: COMPENSATION."
    "38 + 19. 19 is close to 20. 38 + 20 = 58. But I added 1 too many — subtract 1."
    "38 + 20 - 1 = 58 - 1 = 57."

    Write:

    ```

    38 + 19

    = 38 + 20 - 1

    = 58 - 1

    = 57

    ```

    "I made one number EASIER to add (by rounding), then compensated by adjusting."
    "What about 56 + 19? 56 + 20 = 76. Subtract 1: 75."
    "And 56 + 9? 56 + 10 = 66. Subtract 1: 65."
    "The compensation strategy works when one number is close to a multiple of 10."

    WE DO (20 min)

    Compensation practice:

  • 47 + 19: 47 + 20 = 67, 67 - 1 = 66
  • 38 + 29: 38 + 30 = 68, 68 - 1 = 67
  • 65 + 19: 65 + 20 = 85, 85 - 1 = 84
  • 57 + 9: 57 + 10 = 67, 67 - 1 = 66

    • Partner challenge: one partner rounds up, the other rounds down (e.g. 38+29: round 29 to 30, add 38+30=68, subtract 1). Swap for 38+19 (round 19 to 20...).

    YOU DO (15 min)

  • Use compensation to solve: a) 46 + 19 = ___ b) 73 + 29 = ___ c) 58 + 9 = ___
  • "For 73 + 29, what did you round? What did you add? What did you subtract?"
  • "When does compensation work best? Give an example from a real problem."
  • EXTENSION: "Use compensation for 238 + 199. Can you adapt the strategy?"

    • Differentiation

  • EAL/D: Compensation steps on desk as checklist; explicitly model each step
  • NEP: Focus on +9 and +19 only; use number line to show the adjustment
  • Gifted: Extension + "What about 38 + 18? Can compensation work going down? 38+20-2=56?"

    • LESSON 3.2 — Adding with Renaming (Bridging Tens)

    Duration: 50 minutes

    Type: Explore

    Resources: MAB blocks, number lines, place value charts

    Learning Intentions

  • I am learning to add when the ones add to 10 or more
  • I am learning to rename 10 ones as 1 ten

    • Success Criteria

  • I can solve 47 + 28 where 7+8=15 (renaming)
  • I can explain what happens when the ones "overflow"

    • I DO (15 min)

    "47 + 28. Using split: 40 + 20 = 60. 7 + 8 = 15. 60 + 15 = ?"
    "But 15 = 10 + 5. 10 ones = 1 ten! So 60 + 10 = 70, then +5 = 75."
    "We RENAMED 15 ones as 1 ten and 5 ones."

    Build with MAB: 4 tens + 7 ones, add 2 tens + 8 ones.

    "7 ones + 8 ones = 15 ones. Rename: 15 ones = 1 ten + 5 ones."
    "Now we have 5 tens (4+1) and 5 ones. 55."

    WE DO (20 min)

    Renaming practice with MAB:

  • 36 + 28: 6+8=14 → rename 14 ones as 1 ten + 4 ones → 50+4=54
  • 47 + 35: 7+5=12 → rename → 70+12=82
  • 58 + 27: 8+7=15 → rename → 70+15=85
  • 29 + 46: 9+6=15 → rename → 60+15=75

    • Partner: build with MAB, rename, record.

    YOU DO (15 min)

  • Use MAB or draw to solve: a) 38 + 27 = ___ b) 49 + 36 = ___ c) 57 + 28 = ___
  • "For 49 + 36, what is renamed? Why?"
  • "Show 38 + 27 without MAB. How do you track the renaming?"
  • EXTENSION: "What is the smallest 2-digit + 2-digit problem that needs renaming? What is the largest that doesn't need renaming?"

    • Differentiation

  • EAL/D: MAB blocks essential; renaming steps drawn step-by-step
  • NEP: Focus on cases where ones sum 10-12 only (e.g. 47+23, 38+32); use MAB for all
  • Gifted: Extension + "What about 3-digit renaming: 156+237?"

    • LESSON 3.3 — Mixed Strategy Practice

    Duration: 50 minutes

    Type: Guided Practice

    Resources: Problem cards, MAB blocks, number lines

    Learning Intentions

  • I am learning to choose the most efficient strategy for each addition problem
  • I am becoming fluent with 2-digit addition

    • Success Criteria

  • I can add 2-digit numbers using jump, split, or compensation
  • I can explain my strategy choice

    • MIXED PRACTICE (40 min)

    Four stations, 10 min each:

    Station 1 — Strategy Challenge:

    ```

    52

  • 27

    • ```

  • 27

    • ```

  • 27

    ```

    Problem-solving strategy (SOLVE):

  • 247
